# 画像仕様

## 1. 保持単位

- 画像は `Sake` に紐付ける
- 1酒につき 0..N 件
- 保存項目は `imageUris: List<String>`

---

## 2. 取り扱い

- URIでのみ管理し、直接ファイルパスを前提にしない
- 画像が未設定でも酒保存を許可する
- 登録元は酒編集画面とする
- 酒編集画面では複数画像 preview を表示する
- 酒一覧画面では 2列カードで画像 preview を表示してよい
- 酒一覧画面の画像 preview は `setting_image_preview` が有効なときだけ表示する
- 酒一覧画面で画像未登録の酒は placeholder を表示する
- 画像削除は確認ダイアログを経由させる
- 画像の追加・差し替え・削除は保存時に確定させる
- ギャラリーから選んだ画像は保存時にアプリ専用領域へコピーし、永続 URI として扱う
- カメラ撮影画像は編集中は一時 URI として扱い、保存時にアプリ専用領域へ取り込む
- 保存後の画像 URI はアプリ管理領域の URI を保持し、アプリが所有する
- 画像差し替え時と酒削除時の既定動作は、DB の参照だけを更新し、旧 app-managed 画像は即削除しない
- 設定画面に `未参照アプリ内画像を削除` を置き、未参照 app-managed 画像はユーザ操作で cleanup できるようにする
- 設定 `自動画像削除` が ON のときだけ、酒削除時と保存時の画像差し替え後に未参照 app-managed 画像 cleanup を自動実行する
- cleanup 対象は app-managed directory 配下の未参照画像のみとし、外部 URI は削除対象にしない
- 画像表示は S5（画像ビューア）で行う
